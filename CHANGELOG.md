2026-05-17 - 3.0.0
* Major refactor for ClusterControl 2.4.x compatibility.
* Removed Apache/PHP dependency entirely — uses cmon-proxy (MCC mode) as the web server.
* Split into focused per-task classes: install::redhat, install::debian, configure_mysql, configure_mcc, and mcc.
* Idempotent initialization via marker files in /var/lib/cmon (cmon --init and ccmgradm init each run exactly once).
* RPC token generated once and persisted to disk under /var/lib/cmon/cc_rpc_token.
* Uses MySQL Community 8.4 LTS from MySQL's official repo (no MariaDB).
* mysql_native_password auth plugin enforced for ClusterControl compatibility.
* Idempotent cmon user creation per host (localhost, 127.0.0.1, and controller IP).
* ccsetup user created without --email-address so the GUI registration page email field stays editable.
* Admin user is automatically unsuspended before ccsetup creation to handle re-runs cleanly.
* ccmgradm init handles the 'controller already exists' case gracefully.
* Supported OS expanded to: RHEL/CentOS/Rocky/AlmaLinux/OracleLinux 7-9, Ubuntu 18/20/22/24, Debian 9/10/11/12.
* Requires Puppet 7.0+ (uses structured $facts[] hash); compatible with Puppet 8.x.
* Breaking change: parameter set is simplified. Only mysql_root_password and cmon_mysql_password are now required. Many legacy parameters removed (is_controller, cc_hostname, api_token, ssh_user, only_cc_v2, etc.). See README.md for the new parameter list.

2024-04-29 - 2.0.0
* Deploys the latest version of ClusterControl 1.9.8 version. 
* Allows you to deploy ClusterControl version 1 (CCv1) and ClusterControl version 2 (CCv2) or just CCv2 (default). CCv2 does not depend anymore on PHP.
* since CCv1 does not support PHP 8.x versions, this deployment automates the setup for latest OS like RHEL 9 or Ubuntu 22.04 (Jammy)
* fixes issues with controller_id in /etc/cmon.cnf causing restarting of cmon, create SSLs also to be generated by this ClusterControl Puppet module, PHP is no longer installed when only CCv2 is deployed.
* tested to work on EL 9, Debian 11/Ubuntu 22.04, and SLES 15.x. Other latest OS not mentioned are not yet supported by ClusterControl.
